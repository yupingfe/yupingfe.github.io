# JavaScript 标准风格指南

这是 JavaScript standardopen in new window 代码规范的全文。

  • 使用两个空格进行缩进。

    eslint: indentopen in new window

    function hello (name) {
  console.log('hi', name)
}

  • 除需要转义的情况外，字符串统一使用单引号。

    eslint: quotesopen in new window

    console.log('hello there')
$("<div class='box'>")

  • 不要定义未使用的变量。

    eslint: no-unused-varsopen in new window

    function myFunction () {
  var result = something()   // ✗ avoid
}

  • 关键字后面加空格。

    eslint: keyword-spacingopen in new window

    if (condition) { ... }   // ✓ ok
if(condition) { ... }    // ✗ avoid

  • 函数声明时括号与函数名间加空格。

    eslint: space-before-function-parenopen in new window

    function name (arg) { ... }   // ✓ ok
function name(arg) { ... }    // ✗ avoid

run(function () { ... })      // ✓ ok
run(function() { ... })       // ✗ avoid

  • 始终使用 === 替代 ==。
    例外： obj == null 可以用来检查 null || undefined。

    eslint: eqeqeqopen in new window

    if (name === 'John')   // ✓ ok
if (name == 'John')    // ✗ avoid

    if (name !== 'John')   // ✓ ok
if (name != 'John')    // ✗ avoid

  • 字符串拼接操作符 (Infix operators) 之间要留空格。

    eslint: space-infix-opsopen in new window

    // ✓ ok
var x = 2
var message = 'hello, ' + name + '!'

    // ✗ avoid
var x=2
var message = 'hello, '+name+'!'

  • 逗号后面加空格。

    eslint: comma-spacingopen in new window

    // ✓ ok
var list = [1, 2, 3, 4]
function greet (name, options) { ... }

    // ✗ avoid
var list = [1,2,3,4]
function greet (name,options) { ... }

  • else 关键字要与花括号保持在同一行。

    eslint: brace-styleopen in new window

    // ✓ ok
if (condition) {
  // ...
} else {
  // ...
}

    // ✗ avoid
if (condition)
{
  // ...
}
else
{
  // ...
}

  • 多行 if 语句的的括号不能省。

    eslint: curlyopen in new window

    // ✓ ok
if (options.quiet !== true) console.log('done')

    // ✓ ok
if (options.quiet !== true) {
  console.log('done')
}

    // ✗ avoid
if (options.quiet !== true)
  console.log('done')

  • 不要丢掉异常处理中err参数。

    eslint: handle-callback-erropen in new window

    // ✓ ok
run(function (err) {
  if (err) throw err
  window.alert('done')
})

    // ✗ avoid
run(function (err) {
  window.alert('done')
})

  • 使用浏览器全局变量时加上 window. 前缀。
    document、console 和 navigator 除外。

    eslint: no-undefopen in new window

    window.alert('hi')   // ✓ ok

  • 不允许有连续多行空行。

    eslint: no-multiple-empty-linesopen in new window

    // ✓ ok
var value = 'hello world'
console.log(value)

    // ✗ avoid
var value = 'hello world'


console.log(value)

  • 对于三元运算符 ? 和 : 与他们所负责的代码处于同一行

    eslint: operator-linebreakopen in new window

    // ✓ ok
var location = env.development ? 'localhost' : 'www.api.com'

// ✓ ok
var location = env.development
  ? 'localhost'
  : 'www.api.com'

// ✗ avoid
var location = env.development ?
  'localhost' :
  'www.api.com'

  • 每个 var 关键字单独声明一个变量。

    eslint: one-varopen in new window

    // ✓ ok
var silent = true
var verbose = true

// ✗ avoid
var silent = true, verbose = true

// ✗ avoid
var silent = true,
    verbose = true

  • 条件语句中赋值语句使用括号包起来。这样使得代码更加清晰可读，而不会认为是将条件判断语句的全等号（===）错写成了等号（=）。

    eslint: no-cond-assignopen in new window

    // ✓ ok
while ((m = text.match(expr))) {
  // ...
}

// ✗ avoid
while (m = text.match(expr)) {
  // ...
}

  • 单行代码块两边加空格。

    eslint: block-spacingopen in new window

      function foo () {return true}    // ✗ avoid
  function foo () { return true }  // ✓ ok

  • 对于变量和函数名统一使用驼峰命名法。

    eslint: camelcaseopen in new window

      function my_function () { }    // ✗ avoid
  function myFunction () { }     // ✓ ok

  var my_var = 'hello'           // ✗ avoid
  var myVar = 'hello'            // ✓ ok

  • 不允许有多余的行末逗号。

    eslint: comma-dangleopen in new window

      var obj = {
    message: 'hello',   // ✗ avoid
  }

  • 始终将逗号置于行末。

    eslint: comma-styleopen in new window

      var obj = {
    foo: 'foo'
    ,bar: 'bar'   // ✗ avoid
  }

  var obj = {
    foo: 'foo',
    bar: 'bar'   // ✓ ok
  }

  • 点号操作符须与属性需在同一行。

    eslint: dot-locationopen in new window

      console.
    log('hello')  // ✗ avoid

  console
    .log('hello') // ✓ ok

  • 文件末尾留一空行。

    eslint: eol-lastopen in new window

  • 函数调用时标识符与括号间不留间隔。

    eslint: func-call-spacingopen in new window

    console.log ('hello') // ✗ avoid
console.log('hello')  // ✓ ok

  • 键值对当中冒号与值之间要留空白。

    eslint: key-spacingopen in new window

    var obj = { 'key' : 'value' }    // ✗ avoid
var obj = { 'key' :'value' }     // ✗ avoid
var obj = { 'key':'value' }      // ✗ avoid
var obj = { 'key': 'value' }     // ✓ ok

  • 构造函数要以大写字母开头。

    eslint: new-capopen in new window

    function animal () {}
var dog = new animal()    // ✗ avoid

function Animal () {}
var dog = new Animal()    // ✓ ok

  • 无参的构造函数调用时要带上括号。

    eslint: new-parensopen in new window

    function Animal () {}
var dog = new Animal    // ✗ avoid
var dog = new Animal()  // ✓ ok

  • 对象中定义了存值器，一定要对应的定义取值器。

    eslint: accessor-pairsopen in new window

    var person = {
  set name (value) {    // ✗ avoid
    this._name = value
  }
}

var person = {
  set name (value) {
    this._name = value
  },
  get name () {         // ✓ ok
    return this._name
  }
}

  • 子类的构造器中一定要调用 super

    eslint: constructor-superopen in new window

    class Dog {
  constructor () {
    super()   // ✗ avoid
  }
}

class Dog extends Mammal {
  constructor () {
    super()   // ✓ ok
  }
}

  • 使用数组字面量而不是构造器。

    eslint: no-array-constructoropen in new window

    var nums = new Array(1, 2, 3)   // ✗ avoid
var nums = [1, 2, 3]            // ✓ ok

  • 避免使用 arguments.callee 和 arguments.caller。

    eslint: no-calleropen in new window

    function foo (n) {
  if (n <= 0) return

  arguments.callee(n - 1)   // ✗ avoid
}

function foo (n) {
  if (n <= 0) return

  foo(n - 1)
}

  • 避免对类名重新赋值。

    eslint: no-class-assignopen in new window

    class Dog {}
Dog = 'Fido'    // ✗ avoid

  • 避免修改使用 const 声明的变量。

    eslint: no-const-assignopen in new window

    const score = 100
score = 125       // ✗ avoid

  • 避免使用常量作为条件表达式的条件（循环语句除外）。

    eslint: no-constant-conditionopen in new window

    if (false) {    // ✗ avoid
  // ...
}

if (x === 0) {  // ✓ ok
  // ...
}

while (true) {  // ✓ ok
  // ...
}

  • 正则中不要使用控制符。

    eslint: no-control-regexopen in new window

    var pattern = /\x1f/    // ✗ avoid
var pattern = /\x20/    // ✓ ok

  • 不要使用 debugger。

    eslint: no-debuggeropen in new window

    function sum (a, b) {
  debugger      // ✗ avoid
  return a + b
}

  • 不要对变量使用 delete 操作。

    eslint: no-delete-varopen in new window

    var name
delete name     // ✗ avoid

  • 不要定义冗余的函数参数。

    eslint: no-dupe-argsopen in new window

    function sum (a, b, a) {  // ✗ avoid
  // ...
}

function sum (a, b, c) {  // ✓ ok
  // ...
}

  • 类中不要定义冗余的属性。

    eslint: no-dupe-class-membersopen in new window

    class Dog {
  bark () {}
  bark () {}    // ✗ avoid
}

  • 对象字面量中不要定义重复的属性。

    eslint: no-dupe-keysopen in new window

    var user = {
  name: 'Jane Doe',
  name: 'John Doe'    // ✗ avoid
}

  • switch 语句中不要定义重复的 case 分支。

    eslint: no-duplicate-caseopen in new window

    switch (id) {
  case 1:
    // ...
  case 1:     // ✗ avoid
}

  • 同一模块有多个导入时一次性写完。

    eslint: no-duplicate-importsopen in new window

    import { myFunc1 } from 'module'
import { myFunc2 } from 'module'          // ✗ avoid

import { myFunc1, myFunc2 } from 'module' // ✓ ok

  • 正则中不要使用空字符。

    eslint: no-empty-character-classopen in new window

    const myRegex = /^abc[]/      // ✗ avoid
const myRegex = /^abc[a-z]/   // ✓ ok

  • 不要解构空值。

    eslint: no-empty-patternopen in new window

    const { a: {} } = foo         // ✗ avoid
const { a: { b } } = foo      // ✓ ok

  • 不要使用 eval()。

    eslint: no-evalopen in new window

    eval( "var result = user." + propName ) // ✗ avoid
var result = user[propName]             // ✓ ok

  • catch 中不要对错误重新赋值。

    eslint: no-ex-assignopen in new window

    try {
  // ...
} catch (e) {
  e = 'new value'             // ✗ avoid
}

try {
  // ...
} catch (e) {
  const newVal = 'new value'  // ✓ ok
}

  • 不要扩展原生对象。

    eslint: no-extend-nativeopen in new window

    Object.prototype.age = 21     // ✗ avoid

  • 避免多余的函数上下文绑定。

    eslint: no-extra-bindopen in new window

    const name = function () {
  getName()
}.bind(user)    // ✗ avoid

const name = function () {
  this.getName()
}.bind(user)    // ✓ ok

  • 避免不必要的布尔转换。

    eslint: no-extra-boolean-castopen in new window

    const result = true
if (!!result) {   // ✗ avoid
  // ...
}

const result = true
if (result) {     // ✓ ok
  // ...
}

  • 不要使用多余的括号包裹函数。

    eslint: no-extra-parensopen in new window

    const myFunc = (function () { })   // ✗ avoid
const myFunc = function () { }     // ✓ ok

  • switch 一定要使用 break 来将条件分支正常中断。

    eslint: no-fallthroughopen in new window

    switch (filter) {
  case 1:
    doSomething()    // ✗ avoid
  case 2:
    doSomethingElse()
}

switch (filter) {
  case 1:
    doSomething()
    break           // ✓ ok
  case 2:
    doSomethingElse()
}

switch (filter) {
  case 1:
    doSomething()
    // fallthrough  // ✓ ok
  case 2:
    doSomethingElse()
}

  • 不要省去小数点前面的0。

    eslint: no-floating-decimalopen in new window

    const discount = .5      // ✗ avoid
const discount = 0.5     // ✓ ok

  • 避免对声明过的函数重新赋值。

    eslint: no-func-assignopen in new window

    function myFunc () { }
myFunc = myOtherFunc    // ✗ avoid

  • 不要对全局只读对象重新赋值。

    eslint: no-global-assignopen in new window

    window = {}     // ✗ avoid

  • 注意隐式的 eval()。

    eslint: no-implied-evalopen in new window

    setTimeout("alert('Hello world')")                   // ✗ avoid
setTimeout(function () { alert('Hello world') })     // ✓ ok

  • 嵌套的代码块中禁止再定义函数。

    eslint: no-inner-declarationsopen in new window

    if (authenticated) {
  function setAuthUser () {}    // ✗ avoid
}

  • 不要向 RegExp 构造器传入非法的正则表达式。

    eslint: no-invalid-regexpopen in new window

    RegExp('[a-z')    // ✗ avoid
RegExp('[a-z]')   // ✓ ok

  • 不要使用非法的空白符。

    eslint: no-irregular-whitespaceopen in new window

    function myFunc () /*<NBSP>*/{}   // ✗ avoid

  • 禁止使用 __iterator__。

    eslint: no-iteratoropen in new window

    Foo.prototype.__iterator__ = function () {}   // ✗ avoid

  • 外部变量不要与对象属性重名。

    eslint: no-label-varopen in new window

    var score = 100
function game () {
  score: while (true) {      // ✗ avoid
    score -= 10
    if (score > 0) continue score
    break
  }
}

  • 不要使用标签语句。

    eslint: no-labelsopen in new window

    label:
  while (true) {
    break label     // ✗ avoid
  }

  • 不要书写不必要的嵌套代码块。

    eslint: no-lone-blocksopen in new window

    function myFunc () {
  {                   // ✗ avoid
    myOtherFunc()
  }
}

function myFunc () {
  myOtherFunc()       // ✓ ok
}

  • 不要混合使用空格与制表符作为缩进。

    eslint: no-mixed-spaces-and-tabsopen in new window

  • 除了缩进，不要使用多个空格。

    eslint: no-multi-spacesopen in new window

    const id =    1234    // ✗ avoid
const id = 1234       // ✓ ok

  • 不要使用多行字符串。

    eslint: no-multi-stropen in new window

    const message = 'Hello \
                 world'     // ✗ avoid

  • new 创建对象实例后需要赋值给变量。

    eslint: no-newopen in new window

    new Character()                     // ✗ avoid
const character = new Character()   // ✓ ok

  • 禁止使用 Function 构造器。

    eslint: no-new-funcopen in new window

    var sum = new Function('a', 'b', 'return a + b')    // ✗ avoid

  • 禁止使用 Object 构造器。

    eslint: no-new-objectopen in new window

    let config = new Object()   // ✗ avoid

  • 禁止使用 new require。

    eslint: no-new-requireopen in new window

    const myModule = new require('my-module')    // ✗ avoid

  • 禁止使用 Symbol 构造器。

    eslint: no-new-symbolopen in new window

    const foo = new Symbol('foo')   // ✗ avoid

  • 禁止使用原始包装器。

    eslint: no-new-wrappersopen in new window

    const message = new String('hello')   // ✗ avoid

  • 不要将全局对象的属性作为函数调用。

    eslint: no-obj-callsopen in new window

    const math = Math()   // ✗ avoid

  • 不要使用八进制字面量。

    eslint: no-octalopen in new window

    const octal = 042         // ✗ avoid
const decimal = 34        // ✓ ok
const octalString = '042' // ✓ ok

  • 字符串字面量中也不要使用八进制转义字符。

    eslint: no-octal-escapeopen in new window

    const copyright = 'Copyright \251'  // ✗ avoid

  • 使用 __dirname 和 __filename 时尽量避免使用字符串拼接。

    eslint: no-path-concatopen in new window

    const pathToFile = __dirname + '/app.js'            // ✗ avoid

  • 使用 getPrototypeOf 来替代 __proto__。

    eslint: no-protoopen in new window

    const foo = obj.__proto__               // ✗ avoid
const foo = Object.getPrototypeOf(obj)  // ✓ ok

  • 不要重复声明变量。

    eslint: no-redeclareopen in new window

    let name = 'John'
let name = 'Jane'     // ✗ avoid

let name = 'John'
name = 'Jane'         // ✓ ok

  • 正则中避免使用多个空格。

    eslint: no-regex-spacesopen in new window

    const regexp = /test   value/   // ✗ avoid

const regexp = /test {3}value/  // ✓ ok
const regexp = /test value/     // ✓ ok

  • return 语句中的赋值必需有括号包裹。

    eslint: no-return-assignopen in new window

    function sum (a, b) {
  return result = a + b     // ✗ avoid
}

function sum (a, b) {
  return (result = a + b)   // ✓ ok
}

  • 避免将变量赋值给自己。

    eslint: no-self-assignopen in new window

    name = name   // ✗ avoid

  • 避免将变量与自己进行比较操作。

    esint: no-self-compareopen in new window

    if (score === score) {}   // ✗ avoid

  • 避免使用逗号操作符。

    eslint: no-sequencesopen in new window

    if (doSomething(), !!test) {}   // ✗ avoid

  • 不要随意更改关键字的值。

    eslint: no-shadow-restricted-namesopen in new window

    let undefined = 'value'     // ✗ avoid

  • 禁止使用稀疏数组（Sparse arrays）。

    eslint: no-sparse-arraysopen in new window

    let fruits = ['apple',, 'orange']       // ✗ avoid

  • 不要使用制表符。

    eslint: no-tabsopen in new window

  • 正确使用 ES6 中的字符串模板。

    eslint: no-template-curly-in-stringopen in new window

    const message = 'Hello ${name}'   // ✗ avoid
const message = `Hello ${name}`   // ✓ ok

  • 使用 this 前请确保 super() 已调用。

    eslint: no-this-before-superopen in new window

    class Dog extends Animal {
  constructor () {
    this.legs = 4     // ✗ avoid
    super()
  }
}

  • 用 throw 抛错时，抛出 Error 对象而不是字符串。

    eslint: no-throw-literalopen in new window

    throw 'error'               // ✗ avoid
throw new Error('error')    // ✓ ok

  • 行末不留空格。

    eslint: no-trailing-spacesopen in new window

  • 不要使用 undefined 来初始化变量。

    eslint: no-undef-initopen in new window

    let name = undefined    // ✗ avoid

let name
name = 'value'          // ✓ ok

  • 循环语句中注意更新循环变量。

    eslint: no-unmodified-loop-conditionopen in new window

    for (let i = 0; i < items.length; j++) {...}    // ✗ avoid
for (let i = 0; i < items.length; i++) {...}    // ✓ ok

  • 如果有更好的实现，尽量不要使用三元表达式。

    eslint: no-unneeded-ternaryopen in new window

    let score = val ? val : 0     // ✗ avoid
let score = val || 0          // ✓ ok

  • return，throw，continue 和 break 后不要再跟代码。

    eslint: no-unreachableopen in new window

    function doSomething () {
  return true
  console.log('never called')     // ✗ avoid
}

  • finally 代码块中不要再改变程序执行流程。

    eslint: no-unsafe-finallyopen in new window

    try {
  // ...
} catch (e) {
  // ...
} finally {
  return 42     // ✗ avoid
}

  • 关系运算符的左值不要做取反操作。

    eslint: no-unsafe-negationopen in new window

    if (!key in obj) {}       // ✗ avoid

  • 避免不必要的 .call() 和 .apply()。

    eslint: no-useless-callopen in new window

    sum.call(null, 1, 2, 3)   // ✗ avoid

  • 避免使用不必要的计算值作对象属性。

    eslint: no-useless-computed-keyopen in new window

    const user = { ['name']: 'John Doe' }   // ✗ avoid
const user = { name: 'John Doe' }       // ✓ ok

  • 禁止多余的构造器。

    eslint: no-useless-constructoropen in new window

    class Car {
  constructor () {      // ✗ avoid
  }
}

  • 禁止不必要的转义。

    eslint: no-useless-escapeopen in new window

    let message = 'Hell\o'  // ✗ avoid

  • import, export 和解构操作中，禁止赋值到同名变量。

    eslint: no-useless-renameopen in new window

    import { config as config } from './config'     // ✗ avoid
import { config } from './config'               // ✓ ok

  • 属性前面不要加空格。

    eslint: no-whitespace-before-propertyopen in new window

    user .name      // ✗ avoid
user.name       // ✓ ok

  • 禁止使用 with。

    eslint: no-withopen in new window

    with (val) {...}    // ✗ avoid

  • 对象属性换行时注意统一代码风格。

    eslint: object-property-newlineopen in new window

    const user = {
  name: 'Jane Doe', age: 30,
  username: 'jdoe86'            // ✗ avoid
}

const user = { name: 'Jane Doe', age: 30, username: 'jdoe86' }    // ✓ ok

const user = {
  name: 'Jane Doe',
  age: 30,
  username: 'jdoe86'
}                                                                 // ✓ ok

  • 代码块中避免多余留白。

    eslint: padded-blocksopen in new window

    if (user) {
                            // ✗ avoid
  const name = getName()

}

if (user) {
  const name = getName()    // ✓ ok
}

  • 展开运算符与它的表达式间不要留空白。

    eslint: rest-spread-spacingopen in new window

    fn(... args)    // ✗ avoid
fn(...args)     // ✓ ok

  • 遇到分号时空格要后留前不留。

    eslint: semi-spacingopen in new window

    for (let i = 0 ;i < items.length ;i++) {...}    // ✗ avoid
for (let i = 0; i < items.length; i++) {...}    // ✓ ok

  • 代码块首尾留空格。

    eslint: space-before-blocksopen in new window

    if (admin){...}     // ✗ avoid
if (admin) {...}    // ✓ ok

  • 圆括号间不留空格。

    eslint: space-in-parensopen in new window

    getName( name )     // ✗ avoid
getName(name)       // ✓ ok

  • 一元运算符后面跟一个空格。

    eslint: space-unary-opsopen in new window

    typeof!admin        // ✗ avoid
typeof !admin        // ✓ ok

  • 注释首尾留空格。

    eslint: spaced-commentopen in new window

    //comment           // ✗ avoid
// comment          // ✓ ok

/*comment*/         // ✗ avoid
/* comment */       // ✓ ok
